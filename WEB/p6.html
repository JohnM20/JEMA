Objetos literales

Los objetos literales nos permiten guardar elementos partiendo de la descripción de los mismos a continuación se presenta un ejemplo:

    

// Creación de un objeto literal
let persona = {
    nombre: "Juan",
    edad: 30,
    ciudad: "Madrid",
    profesion: "Ingeniero"
};

// Acceder a las propiedades del objeto
console.log("Nombre:", persona.nombre); // Salida: Juan
console.log("Edad:", persona.edad); // Salida: 30

// Modificar una propiedad del objeto
persona.edad = 35;
console.log("Edad modificada:", persona.edad); // Salida: 35

// Añadir una nueva propiedad al objeto
persona.telefono = "123-456-789";
console.log("Teléfono:", persona.telefono); // Salida: 123-456-789

// Eliminar una propiedad del objeto
delete persona.profesion;
console.log("Profesión eliminada:", persona.profesion); // Salida: undefined

// Mostrar todo el objeto
console.log("Objeto completo:", persona);

Uso del Operador de Spread en JavaScript

¿Qué es el Operador de Spread?

El operador de spread en JavaScript, representado por ..., se utiliza para expandir los elementos de un iterable (como arreglos u objetos) en contextos donde se esperan varios elementos. Facilita la manipulación de arreglos y objetos sin necesidad de recurrir a técnicas más complejas.

Uso del Spread con Arreglos

Copiar un Arreglo: El operador de spread permite crear una copia superficial de un arreglo. Esto significa que puedes duplicar un arreglo sin modificar el original. 

let numerosOriginales = [1, 2, 3];
let numerosCopiados = [...numerosOriginales];

console.log(numerosCopiados); // [1, 2, 3]

Concatenar Arreglos: Puedes usar el operador de spread para combinar varios arreglos en uno solo, sin modificar los arreglos originales. 

let arreglo1 = [1, 2, 3];
let arreglo2 = [4, 5, 6];
let arregloConcatenado = [...arreglo1, ...arreglo2];

console.log(arregloConcatenado); // [1, 2, 3, 4, 5, 6]

Uso del Spread con Objetos

Copiar un Objeto: El operador de spread permite crear una copia superficial de un objeto. Esto es útil para evitar modificar el objeto original al hacer cambios. 

let personaOriginal = { nombre: "Alice", edad: 30 };
let personaCopiada = { ...personaOriginal };

console.log(personaCopiada); // { nombre: "Alice", edad: 30 }

Combinar Objetos: Puedes combinar varias propiedades de objetos en un nuevo objeto sin alterar los objetos originales. 

let persona = { nombre: "Alice", edad: 30 };
let direccion = { ciudad: "Wonderland", pais: "Fantasia" };
let personaCompleta = { ...persona, ...direccion };

console.log(personaCompleta);
// { nombre: "Alice", edad: 30, ciudad: "Wonderland", pais: "Fantasia" }

¿Cómo Evita la Duplicación de Objetos?

El operador de spread no evita la duplicación de objetos en el sentido de la memoria, pero facilita la creación de nuevas instancias con los datos deseados. Al usar ... para copiar o combinar objetos, creas nuevas instancias con los datos que puedes manipular independientemente, lo que ayuda a gestionar datos sin alterar los originales.

Por ejemplo, al copiar un objeto con ..., la copia es un nuevo objeto con las mismas propiedades, pero cualquier cambio en la copia no afecta al objeto original. Esto permite trabajar con datos de manera más segura y organizada.


Programación orientada a objetos


Cuando un objeto literal se emplea multiples veces con diferentes caracteristicas se emplea la programación orientada a objetos, a continuación se presenta un ejemplo:


    

// Definición de una clase llamada Persona
class Persona {
    // Constructor de la clase Persona
    constructor(nombre, edad) {
        this.nombre = nombre;
        this.edad = edad;
    }

    // Método para obtener la descripción de la persona
    descripcion() {
        return `${this.nombre} tiene ${this.edad} años.`;
    }
}

// Creación de una instancia de la clase Persona
let persona1 = new Persona("Juan", 30);

// Llamada al método descripcion() de la instancia persona1
console.log(persona1.descripcion()); // Salida: Juan tiene 30 años.
